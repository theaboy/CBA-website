  <action>
    Both components are 'use client' and follow the pattern from RESEARCH.md Pattern 8.

    **components/checkout/BeatCheckout.tsx:**
    Props: { beatId: string; licenseType: 'basic' | 'premium' | 'exclusive'; price: number; buyerEmail?: string; buyerName?: string }

    State: clientSecret (string | null), buyerEmail (string), buyerName (string), loading (boolean), error (string | null)

    UX flow:
    1. Render a two-field form: buyer name input + buyer email input
    2. On "Proceed to Payment" click: call create-payment-intent API, set clientSecret
    3. While loading: show "Setting up payment..." disabled state
    4. Once clientSecret is set: show Elements provider with PaymentElement
    5. Submit button: "Pay ${price} CAD" — disabled while Stripe is loading

    Stripe setup:
    ```tsx
    import { stripePromise } from '@/lib/stripe/client'
    import { Elements, PaymentElement, useStripe, useElements } from '@stripe/react-stripe-js'

    <Elements stripe={stripePromise} options={{ clientSecret }}>
      <CheckoutForm returnUrl="/beats/confirmation" />
    </Elements>
    ```

    CheckoutForm (inner component):
    - useStripe(), useElements()
    - handleSubmit: await stripe.confirmPayment({ elements, confirmParams: { return_url: window.location.origin + returnUrl } })
    - On error from Stripe: display error.message below the PaymentElement
    - On success: Stripe handles redirect to return_url

    Error display: show error message in a styled div. Do not use alert().

    Styling: match site token system — dark card background, CBA accent color for submit button. Keep markup minimal.

    **components/checkout/TicketCheckout.tsx:**
    Props: { eventId: string; eventTitle: string; ticketPrice: number; remainingCapacity: number }

    State: quantity (number, default 1, max = remainingCapacity), clientSecret (string | null), buyerEmail, buyerName, step ('details' | 'payment')

    UX flow:
    1. Step "details": quantity selector (1–4, capped at remainingCapacity), name + email inputs, "Proceed" button
    2. On proceed: call create-payment-intent with { eventId, quantity, buyerEmail, buyerName }, transition to step "payment"
    3. Step "payment": show Elements + PaymentElement + "Pay ${(ticketPrice * quantity).toFixed(2)} CAD" button
    4. returnUrl: "/events/confirmation"

    If remainingCapacity === 0: show "Sold out" text, disable all controls.

    Follow the same Elements + CheckoutForm inner component pattern as BeatCheckout.
  </action>
